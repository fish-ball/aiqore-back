## 文档维护说明

**代码改动时请同步更新文档**，避免文档与实现脱节。

1. **修改或新增功能时**
   - 若影响「做什么、怎么用」：更新 [1-架构与概览](./1-架构与概览.md) 中功能与模块说明、[5-API](./5-API.md) 中接口列表与示例。
   - 若影响「怎么跑起来、依赖、环境」：更新 [2-开发与运行](./2-开发与运行.md)。
   - 若涉及数据库、迁移、编码：更新 [3-数据库与迁移](./3-数据库与迁移.md)。
   - 若涉及前端页面、路由、构建或代理：更新 [4-前端](./4-前端.md)。
   - 若涉及 QMT、证券/行情逻辑：更新 [6-QMT集成](./6-QMT集成.md)。
   - 若涉及 Celery 任务、Redis、异步接口：更新 [7-异步任务](./7-异步任务.md)；**新增或修改异步任务时**，必须更新该文档中的「已实现的异步任务清单」表及对应任务小节。

2. **建议习惯**
   - 提交代码前快速扫一眼相关 doc 是否需要改。
   - 新增 API 或路由时，在 [5-API](./5-API.md) 补充路径与简短说明。
   - 新增后端模块或前端大模块时，在 [1-架构与概览](./1-架构与概览.md) 中补充到结构树与模块职责表。
   - 新增或修改 `app/tasks/` 下 Celery 任务时，在 [7-异步任务](./7-异步任务.md) 中更新「已实现的异步任务清单」表及对应任务小节（参数、触发方式、逻辑、状态与结果等）。

3. **原散落文档**
   - 根目录及 frontend 下的原有 .md（如 README、QMT集成说明、异步任务说明、DATABASE_SETUP、ALEMBIC_ENCODING_FIX、MIGRATION_TO_UV、UV_SETUP、frontend/使用说明、启动说明等）内容已整合到本目录对应文档；保留或删除原文件可由团队自行决定，本目录为当前权威文档来源。
