# 异步任务

## 概述

基于 Celery + Redis 的异步任务系统，用于长时间运行任务（如证券信息全量/分批更新），避免 HTTP 超时。

## 技术栈

- **Celery**：分布式任务队列
- **Redis**：消息代理与结果后端（如 WSL 下 6379 端口）

## 配置

### Redis

在 `app/config.py` 中配置（或通过环境变量）：

```python
REDIS_HOST = "localhost"   # Redis 在 WSL 时可改为 WSL IP
REDIS_PORT = 6379
REDIS_PASSWORD = "redis"
REDIS_DB = 0
```

Redis 在 WSL 时，可从 Windows 使用 WSL IP：在 WSL 中执行 `ip addr show eth0` 或 `hostname -I` 获取 IP，将 `REDIS_HOST` 设为该 IP；或确保 6379 已端口映射。

### 依赖

```bash
uv sync
# 已包含 celery、redis
```

## 启动

### 1. 启动 Redis

WSL 示例：

```bash
wsl redis-server
# 或 Docker
wsl docker run -d --name redis -p 6379:6379 redis:latest redis-server --requirepass redis
```

### 2. 启动 Celery Worker

```bash
uv run celery -A app.celery_app worker --loglevel=info --pool=solo
# 或 Windows 下
start_celery_worker.bat
```

Windows 必须使用 `--pool=solo`。

### 3. 启动 FastAPI

```bash
uv run python run.py
```

## 使用方式

### 前端

- 证券列表页点击「从 QMT 更新」
- 接口返回任务 ID，前端轮询 `GET /api/task/{task_id}` 显示进度，完成后刷新列表

### API

- **提交更新任务**：`POST /api/security/update?market=SH`（可选 market）
- **查询状态**：`GET /api/task/{task_id}`

任务状态：PENDING、PROGRESS（含 current/total/progress）、SUCCESS、FAILURE。

### 任务说明：update_securities_task

- 参数：market（'SH'/'SZ' 或 None 表示全部）
- 逻辑：从 QMT 获取标的、批量获取名称、更新或创建数据库记录、返回统计
- 会周期更新进度，供前端轮询

## 故障排查

- **Celery 连不上 Redis**：检查 Redis 是否运行（如 `wsl redis-cli -a redis ping`）、REDIS_* 配置、若在 WSL 尝试用 WSL IP
- **Worker 启动失败**：确认 celery/redis 已安装，Windows 使用 `--pool=solo`
- **任务一直 PENDING**：确认 Worker 已启动、查看 Worker 日志、确认任务已注册
- **任务执行失败**：查看 Worker 日志（可加 `--loglevel=debug`），检查 QMT 与数据库连接

## 监控（可选）

```bash
pip install flower
uv run celery -A app.celery_app flower
```

访问 http://localhost:5555 查看任务与 Worker 状态。

## 扩展新任务

1. 在 `app/tasks/` 下新增模块，使用 `@celery_app.task(bind=True)` 装饰
2. 在 API 中通过 `task.delay(...)` 提交
3. 使用任务 ID 查询状态（可沿用现有 `/api/task/{task_id}` 或自建）
4. 需要进度时在任务内调用 `self.update_state(state="PROGRESS", meta={...})`

任务由 `app/celery_app.py` 的 `include=["app.tasks"]` 自动发现。
