# 异步任务

## 概述

基于 Celery + Redis 的异步任务系统，用于长时间运行任务（如证券信息全量/分批更新），避免 HTTP 超时。

## 技术栈

- **Celery**：分布式任务队列
- **Redis**：消息代理与结果后端（如 WSL 下 6379 端口）

## 已实现的异步任务清单

当前所有 Celery 任务定义在 `app/tasks/` 下，由 `app/celery_app.py` 的 `include=["app.tasks"]` 自动发现。**新增或修改任务时请同步更新下表及下文对应小节。**

| 任务名称（Celery name） | 函数名 | 所在文件 | 触发方式 | 说明 |
|------------------------|--------|----------|----------|------|
| `update_securities` | `update_securities_task` | `app/tasks/security_tasks.py` | `POST /api/security/update` | 从 QMT 同步证券基础信息 |

---

### 1. update_securities（证券信息同步）

- **Celery 名称**：`update_securities`
- **函数**：`update_securities_task`（`app/tasks/security_tasks.py`）
- **触发接口**：`POST /api/security/update`，query 参数：`market`（可选）、`sector`（可选）
- **参数**：
  - `market`：`'SH'` / `'SZ'` 或 `None`（全部）
  - `sector`：板块名称，若指定则只同步该板块的证券
- **逻辑**：使用 `TaskLock` 防止同一任务并发执行（锁超时 2 小时）；调用 `security_service.update_securities_from_qmt(db, market, sector)` 从 QMT 获取标的并写入数据库。
- **状态与结果**：
  - 启动时：`PROGRESS`，meta 含 `current=0, total=0, status="任务已启动，开始获取股票列表..."`
  - 成功：`SUCCESS`，meta 含 `result`（success、total、created、updated、errors 等）
  - 业务失败（如获取锁失败）：`SUCCESS`，meta 中 result 含 `success: false` 及 message
  - 异常：抛出异常由 Celery 置为 `FAILURE`
- **返回结果字段**：`success`、`message`、`total`、`created`、`updated`、`errors`
- **并发控制**：同一时间仅允许一个 `update_securities` 运行；若已有实例在跑，接口返回 409，任务返回“任务已在运行中”。

## 配置

### Redis

在 `app/config.py` 中配置（或通过环境变量）：

```python
REDIS_HOST = "localhost"   # Redis 在 WSL 时可改为 WSL IP
REDIS_PORT = 6379
REDIS_PASSWORD = "redis"
REDIS_DB = 0
```

Redis 在 WSL 时，可从 Windows 使用 WSL IP：在 WSL 中执行 `ip addr show eth0` 或 `hostname -I` 将 `REDIS_HOST` 设为该 IP；或确保 6379 已端口映射。

### 依赖

```bash
uv sync
# 已包含 celery、redis
```

## 启动

### 1. 启动 Redis

WSL 示例：

```bash
wsl redis-server
# 或 Docker
wsl docker run -d --name redis -p 6379:6379 redis:latest redis-server --requirepass redis
```

### 2. 启动 Celery Worker

```bash
uv run celery -A app.celery_app worker --loglevel=info --pool=solo
# 或 Windows 下
start_celery_worker.bat
```

Windows 必须使用 `--pool=solo`。

### 3. 启动 FastAPI

```bash
uv run python run.py
```

## 使用方式

### 前端

- 证券列表页点击「从 QMT 更新」
- 接口返回任务 ID，前端轮询任务状态接口显示进度，完成后刷新列表

### API

- **提交证券更新任务**：`POST /api/security/update?market=SH&sector=可选板块名`，返回 `task_id`
- **查询任务状态**：需在应用中实现 `GET /api/task/{task_id}` 或通过 Celery 的 `AsyncResult` 查询；任务状态：PENDING、PROGRESS、SUCCESS、FAILURE

## 故障排查

- **Celery 连不上 Redis**：检查 Redis 是否运行（如 `wsl redis-cli -a redis ping`）、REDIS_* 配置、若在 WSL 尝试用 WSL IP
- **Worker 启动失败**：确认 celery/redis 已安装，Windows 使用 `--pool=solo`
- **任务一直 PENDING**：确认 Worker 已启动、查看 Worker 日志、确认任务已注册
- **任务执行失败**：查看 Worker 日志（可加 `--loglevel=debug`），检查 QMT 与数据库连接
- **提示“任务已在运行中”**：同一任务带锁，需等当前实例结束或锁超时（2 小时）后再试

## 扩展新任务

1. 在 `app/tasks/` 下新增或修改任务模块，使用 `@celery_app.task(bind=True, name="任务名")` 装饰
2. 在 API 中通过 `task.delay(...)` 提交
3. 需要时提供或沿用任务状态查询接口（如 `/api/task/{task_id}`）
4. 需要进度时在任务内调用 `self.update_state(state="PROGRESS", meta={...})`
5. **同步更新本文档**：在「已实现的异步任务清单」表中新增一行，并在下方增加对应小节（参数、触发方式、逻辑、状态与结果、并发控制等）。

任务由 `app/celery_app.py` 的 `include=["app.tasks"]` 自动发现。
