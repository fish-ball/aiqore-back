# 数据库与迁移

## 数据库配置

支持 PostgreSQL 与 SQLite。配置通过 `.env` 或 `app/config.py` 中的 DB_* 环境变量。

### WSL Docker PostgreSQL 示例

- 容器名：`postgres`
- 数据库名：`mydb`
- 用户/密码：`postgres` / `postgres`
- 端口：5432

**使用 WSL IP 直连（无需端口映射）示例：**

在 WSL 中查看 IP：`wsl hostname -I`，在 `.env` 中设置：

```
DB_HOST=172.30.37.138
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=postgres
DB_NAME=mydb
```

注意：WSL 重启后 IP 可能变化，若连不上需重新查 IP 并更新 `DB_HOST`。

**验证连接：**

```powershell
Test-NetConnection -ComputerName <DB_HOST> -Port 5432
```

```bash
uv run python -c "from app.database import engine; from sqlalchemy import text; c=engine.connect(); print(c.execute(text('SELECT version()')).fetchone()[0])"
```

### 使用 localhost（需端口映射）

若希望用 localhost，需映射端口后启动容器，例如：

```bash
wsl docker run -d --name postgres -e POSTGRES_DB=mydb -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=postgres -p 5432:5432 postgres:latest
```

然后在 `.env` 中设置 `DB_HOST=localhost`。

## 初始化与迁移

### 初始化表结构

```bash
uv run python init_db.py
```

或使用 Alembic（推荐，支持版本管理）：

```bash
uv run alembic upgrade head
```

### 创建新迁移

```bash
uv run alembic revision --autogenerate -m "描述"
uv run alembic upgrade head
```

## Windows 下 Alembic 编码问题

在 Windows 中文系统上执行 `alembic upgrade head` 或 `alembic revision` 时，可能出现：

```
UnicodeDecodeError: 'gbk' codec can't decode byte 0xa8 in position ...
```

原因：Windows 默认 locale 为 GBK，而 `alembic.ini` 为 UTF-8，Alembic 读配置时使用 locale 编码导致冲突。

### 解决方式

1. **使用包装脚本（推荐）**  
   若项目提供 `alembic_wrapper.py`：
   ```bash
   python alembic_wrapper.py upgrade head
   python alembic_wrapper.py revision --autogenerate -m "message"
   ```

2. **使用 UTF-8 批处理**  
   若提供 `alembic_utf8.bat` 或 `alembic_utf8.ps1`：
   ```bash
   alembic_utf8.bat upgrade head
   # 或 PowerShell: .\alembic_utf8.ps1 upgrade head
   ```

3. **设置环境变量后执行**  
   ```powershell
   $env:PYTHONIOENCODING = "utf-8"
   uv run alembic upgrade head
   ```

4. **确保 `alembic.ini` 以 UTF-8 保存**  
   避免文件中出现非 ASCII 字符在 GBK 下被误读。

修复原理：在导入 Alembic 前或读配置时强制使用 UTF-8（例如猴子补丁 `alembic.util.compat.read_config_parser` 使用 `encoding="utf-8"`）。详见原 ALEMBIC_ENCODING_FIX.md（内容已整合到本文档）。
