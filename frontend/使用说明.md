# 解决 npm 执行策略问题

## 问题说明

Windows PowerShell 默认禁止运行脚本，导致无法执行 npm 命令。

## 解决方案（三种方法）

### 方法 1：修改执行策略（推荐）

**步骤：**

1. **以管理员身份打开 PowerShell**
   - 按 `Win + X`，选择"Windows PowerShell (管理员)" 或 "终端 (管理员)"

2. **运行以下命令：**
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
   
3. **输入 `Y` 确认**

4. **验证设置：**
   ```powershell
   Get-ExecutionPolicy
   ```
   应该显示 `RemoteSigned`

5. **现在可以正常使用 npm：**
   ```powershell
   cd frontend
   npm install
   ```

### 方法 2：使用 CMD 命令行（最简单）

如果不想修改 PowerShell 设置，可以直接使用 CMD：

1. **打开 CMD（命令提示符）**
   - 按 `Win + R`，输入 `cmd`，回车

2. **运行 npm 命令：**
   ```cmd
   cd frontend
   npm install
   ```

### 方法 3：临时绕过执行策略

在 PowerShell 中运行单个命令时临时绕过：

```powershell
powershell -ExecutionPolicy Bypass -Command "cd frontend; npm install"
```

或者：

```powershell
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
cd frontend
npm install
```

## 推荐方案

**推荐使用方法 1**，因为：
- 一次设置，永久有效
- 安全性好（RemoteSigned 只允许本地脚本和已签名的远程脚本）
- 不影响系统其他用户

## 执行策略说明

- **Restricted**：禁止所有脚本（默认）
- **RemoteSigned**：允许本地脚本，远程脚本需要签名（推荐）
- **Unrestricted**：允许所有脚本（不推荐，安全性低）
- **Bypass**：完全绕过（仅临时使用）

## 如果仍然遇到问题

1. **检查 Node.js 是否正确安装：**
   ```powershell
   node -v
   npm -v
   ```

2. **如果命令不存在，请安装 Node.js：**
   - 访问 https://nodejs.org/
   - 下载并安装 LTS 版本

3. **使用 npm 国内镜像（如果下载慢）：**
   ```powershell
   npm config set registry https://registry.npmmirror.com
   ```
