# 异步任务系统说明

## 概述

已实现基于 Celery + Redis 的异步任务系统，用于处理长时间运行的任务（如证券数据更新），避免HTTP请求超时。

## 技术栈

- **Celery**: Python分布式任务队列
- **Redis**: 消息代理和结果后端（运行在WSL的6379端口）

## 配置说明

### Redis配置

Redis运行在WSL的6379端口，密码为`redis`。

配置在 `app/config.py` 中：
```python
REDIS_HOST: str = "localhost"  # 如果Redis在WSL，可能需要使用WSL IP
REDIS_PORT: int = 6379
REDIS_PASSWORD: str = "redis"
REDIS_DB: int = 0
```

### 如果Redis在WSL中

如果从Windows访问WSL中的Redis，可能需要：

1. **使用WSL IP地址**（推荐）
   - 在WSL中运行：`ip addr show eth0` 查看IP
   - 在 `app/config.py` 中设置 `REDIS_HOST` 为WSL IP

2. **端口映射**
   - 确保WSL的6379端口已映射到Windows

## 安装依赖

```bash
pip install celery redis
```

或使用 UV（推荐）：
```bash
uv sync
```

## 启动服务

### 1. 启动Redis（如果还没启动）

如果Redis在WSL中：
```bash
wsl redis-server
```

或使用Docker：
```bash
wsl docker run -d --name redis -p 6379:6379 redis:latest redis-server --requirepass redis
```

### 2. 启动Celery Worker

**方法一：使用批处理文件（Windows）**
```bash
start_celery_worker.bat
```

**方法二：使用命令行**
```bash
celery -A app.celery_app worker --loglevel=info --pool=solo
```

**注意**：在Windows上需要使用 `--pool=solo`，因为Windows不支持进程池。

### 3. 启动FastAPI服务

```bash
python run.py
```

或：
```bash
uvicorn app.main:app --reload
```

## 使用说明

### 前端使用

1. 打开证券列表页面
2. 点击"从QMT更新"按钮
3. 系统会：
   - 立即返回任务ID
   - 在后台异步执行更新任务
   - 前端自动轮询任务状态
   - 显示更新进度
   - 完成后自动刷新列表

### API接口

#### 提交更新任务
```
POST /api/security/update?market=SH
```

返回：
```json
{
  "code": 0,
  "data": {
    "task_id": "xxx-xxx-xxx",
    "status": "PENDING"
  },
  "message": "任务已提交，正在后台处理"
}
```

#### 查询任务状态
```
GET /api/task/{task_id}
```

返回：
```json
{
  "code": 0,
  "data": {
    "task_id": "xxx-xxx-xxx",
    "state": "PROGRESS",
    "status": "处理中",
    "current": 100,
    "total": 5000,
    "progress": 2
  }
}
```

任务状态：
- `PENDING`: 等待中
- `PROGRESS`: 处理中（包含进度信息）
- `SUCCESS`: 完成
- `FAILURE`: 失败

## 任务说明

### update_securities_task

异步更新证券基础信息任务。

**参数**:
- `market`: 市场代码，'SH' 或 'SZ'，None表示全部

**功能**:
- 从QMT获取股票列表
- 批量获取股票名称
- 更新或创建数据库记录
- 返回更新统计

**进度更新**:
- 任务会定期更新进度信息
- 前端可以轮询获取最新状态

## 故障排查

### 问题1: Celery无法连接Redis

**解决方案**:
1. 检查Redis是否运行：`wsl redis-cli -a redis ping`
2. 检查Redis配置是否正确
3. 如果Redis在WSL，尝试使用WSL IP地址

### 问题2: Worker启动失败

**解决方案**:
1. 确保已安装celery和redis：`pip install celery redis`
2. Windows上必须使用 `--pool=solo`
3. 检查Redis连接是否正常

### 问题3: 任务一直处于PENDING状态

**解决方案**:
1. 检查Celery Worker是否正在运行
2. 查看Worker日志了解错误信息
3. 检查任务是否被正确注册

### 问题4: 任务执行失败

**解决方案**:
1. 查看Worker日志：`celery -A app.celery_app worker --loglevel=debug`
2. 检查QMT连接是否正常
3. 检查数据库连接是否正常

## 性能优化建议

1. **批量处理**: 已实现批量获取名称，减少API调用
2. **分批提交**: 每100条记录提交一次，避免事务过大
3. **错误处理**: 单个股票失败不影响整体任务
4. **进度更新**: 定期更新进度，方便前端显示

## 扩展其他异步任务

如果需要添加其他异步任务：

1. 在 `app/tasks/` 目录创建新的任务文件
2. 使用 `@celery_app.task` 装饰器
3. 在API中调用 `task.delay()` 启动任务
4. 使用任务ID查询状态

示例：
```python
from app.celery_app import celery_app

@celery_app.task(bind=True)
def my_task(self, param1, param2):
    # 更新进度
    self.update_state(state="PROGRESS", meta={"current": 10, "total": 100})
    # 执行任务
    # ...
    return result
```
